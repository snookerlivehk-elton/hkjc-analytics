# HKJC Analytics - 獨立計分條件與評分邏輯清單

本文件詳細列出系統中計分條件的所需數據來源、評分邏輯，以及目前的開發狀態。系統透過將各項原始數據標準化為 0-10 分，並依據權重計算總分，以產出每場賽事的專業排名。

## ✅ 第一類：數據已具備，可立即實作真實邏輯（共 10 項）
這些條件所需的資料，我們已經成功從「排位表」和「歷史往績」中抓取並存入資料庫，隨時可以寫程式讓它們生效。

| 條件名稱 | 所需數據來源 | 評分邏輯設計 (演算法) |
| :--- | :--- | :--- |
| **1. 騎師＋練馬師合作** | 歷史往績 (騎師、練馬師、馬匹、名次) | 綜合「全庫合作」與「本駒合作」兩個維度，統計勝率/上名率並依權重加權合併，分數越高越好。 |
| **2. 場地＋路程專長** | 歷史往績 (場地、途程、名次) | 篩選該馬在「今日相同場地與途程」的歷史紀錄，計算其上名率及平均名次。 |
| **3. 檔位偏差** | 歷史往績 (檔位、途程、名次) | 大數據統計：例如谷草 1200m，歷史數據顯示 1-4 檔勝率明顯較高，排內檔者加分。 |
| **4. 負磅／評分表現** | 排位表 (目前評分) + 歷史往績 | 尋找該馬過去「贏馬時的最高評分」。如果目前評分低於其贏馬評分（代表目前佔優），給予加分。 |
| **5. 練馬師＋馬匹組合** | 歷史往績 (馬匹、練馬師、名次) | 統計該練馬師過往訓練本駒的出賽表現（勝率/上名率），可選近 X 次樣本並以權重加權。 |
| **6. 配備變化** | 排位表 (配備) + 歷史往績 (配備) | 檢查今日是否「初戴」或「重戴」某配備（如眼罩 B）。比對其歷史上戴該配備時的成績是否有顯著提升。 |
| **7. 班次表現** | 排位表 (班次) + 歷史往績 (班次) | 判斷是否為「降班戰」（如上仗跑三班，今仗跑四班），降班通常對手變弱，給予大幅加分。 |
| **8. 場地狀況專長** | 排位表 (好/黏地) + 歷史往績 | 若今日為雨天變化地（黏/軟），則檢查該馬歷史上在變化地的勝率，擅長泥沼者加分。 |
| **9. 近期狀態 (Last 6)** | 歷史往績 (最近 6 場名次) | 計算近 6 仗的「加權平均名次」（越近的比賽權重越高）。名次越小、趨勢向上者得高分。 |
| **10. 初出／長休後表現** | 歷史往績 (出賽日期差距) | 計算上仗距今的「休息天數」。若大於 90 天，則比對其歷史上「休後復出」的勝率表現。 |

## ❌ 第二類：目前尚缺數據，需要擴充爬蟲模組（共 6 項）
這些條件需要開發新的爬蟲，去抓取馬會的其他頁面（如即時賠率、晨操紀錄、走位等）才能實作。

| 條件名稱 | 缺少的數據 | 解決方案與邏輯 |
| :--- | :--- | :--- |
| **11. 馬匹分段時間** | 歷史往績缺少「段速」資料 | **需擴充爬蟲**：抓取賽果派彩頁面的「分段時間」，比較該馬的末段衝刺速度 (如 22.5秒) 是否為同組最快。 |
| **12. 投注額變動 (落飛)** | 缺少「即時賠率」與「早盤賠率」 | **需擴充爬蟲**：開發賠率爬蟲。若賽前 10 分鐘獨贏賠率大幅下降（如 10 倍跌至 5 倍），代表有大戶落飛，加分。 |
| **13. 晨操／試閘表現** | 缺少「晨操及試閘紀錄」 | **需擴充爬蟲**：抓取馬匹晨操頁。近期試閘跑第一、或拍跳走勢強勁者加分。 |
| **14. 配速分析 (跑法)** | 歷史往績缺少「走位 (1-1-2)」 | **需擴充爬蟲**：抓取詳細賽果的走位。判斷是領放馬還是後上馬，分析本場賽事步速是否有利其跑法。 |
| **15. HKJC SpeedPRO** | 缺少「SpeedPRO 頁面數據」 | **需擴充爬蟲**：直接抓取馬會官方 SpeedPRO 提供的能量分數與預測。 |
| **16. 獸醫報告** | 缺少「獸醫檢驗紀錄」 | **需擴充爬蟲**：抓取獸醫報告。近期有「氣管多血」、「不良於行」紀錄者大幅扣分。 |
